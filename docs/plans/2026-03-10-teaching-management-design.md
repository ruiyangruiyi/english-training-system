# 教学管理系统 — 考勤/老师管理/登录机制 设计稿 (2026-03-10)

## 背景
当前系统存在以下问题：
- 考勤无法切换日期/历史编辑、保存后状态不一致
- 无老师注册/审核/禁用机制
- 登录逻辑缺失（新设备可直接进入）
- 名称需统一为“教学管理系统”

## 目标（MVP）
1) 考勤可切换日期查看/编辑（历史可改）
2) 考勤状态：出勤/请假/缺勤
3) 请假/缺勤支持可选备注（每学生一条，仅在请假/缺勤显示）
4) 保存时弹窗确认；若无变更提示“无变更”
5) 老师自助注册 → 管理员审核启用；可禁用老师（不删数据）
6) 登录必须校验，支持记住登录 7/30 天
7) 全站名称改为“教学管理系统”

## 方案选型
### 考勤日期存储
- **方案A（采用）**：仍用 `date` 字段，但查询以“当天时间范围”匹配（避免时区导致查不到）
- 方案B：新增 `dateStr`（YYYY-MM-DD）字段（改动更大）

## 数据结构（拟调整）
- Attendance 增加 `remark`（可空）
- User 增加字段：`phone`, `wechat`, `subject`, `status`（pending/active/disabled）, `role`（admin/teacher）
- Auth/session：新增 session 表或使用 jwt token（最终实现需与现有登录接口对齐）

## 功能设计
### 1. 考勤页面
- 日期选择：可自由切换
- 表格：每学生一行，状态按钮
- 当状态为请假/缺勤，显示备注输入框（可空）
- 保存按钮：弹窗确认
- 未改动：提示“无变更”

### 2. 老师注册/审核
- 注册：姓名/手机号/微信号/科目 必填
- 注册后状态为 pending
- 管理员审核通过后才能登录
- 禁用：状态改为 disabled（不可登录，但数据保留）

### 3. 登录机制
- 未登录访问任意页面跳转登录
- 记住登录：7天/30天（前端勾选，后端设置 session 过期）

### 4. 命名统一
- 标题、页脚、登录页、导航栏统一改为“教学管理系统”

## 验收标准（关键点）
- 可切换日期查看并编辑历史考勤
- 保存后再次进入，状态与备注保持
- 请假/缺勤才出现备注输入
- 无变更点击保存提示“无变更”
- 老师注册后需管理员审核才能登录
- 禁用老师不可登录，数据保留可查看
- 新设备必须登录
- 全站名称替换完成

## 风险
- 现有登录机制缺失，需补充 session/鉴权（需验证现有接口）
